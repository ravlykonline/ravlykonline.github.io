# TASKS.md — практичний backlog реалізації Ravlyk Artist

Цей документ — робочий список задач. Він має використовуватися як технічне завдання для Codex/Claude Code. Задачі розбиті за пріоритетами.

---

## P0 — стабілізація фундаменту

Ці задачі виконати до будь-якого великого нарощування функціоналу.

### P0.1. Ізолювати legacy-код

- [ ] Створити папку `legacy/`.
- [ ] Перенести `script.js` у `legacy/script.legacy.js`.
- [ ] Перенести `style.css` у `legacy/style.legacy.css`.
- [ ] Перевірити, що `index.html` не посилається на legacy-файли.
- [ ] Додати пояснення в `README.md`.

Критерій готовності: у корені немає активних старих файлів, агенти не плутають production-код.

---

### P0.2. Додати package.json

- [ ] Додати `package.json`.
- [ ] Вказати `"type": "module"`.
- [ ] Додати script `test`.
- [ ] Додати script `validate:levels` після створення validator.
- [ ] Додати script `lint` після налаштування ESLint.

Критерій готовності:

```bash
npm test
```

запускає поточні тести.

---

### P0.3. Перекласти UI українською

Файли:

- `index.html`;
- `js/data/blocks.js`;
- `js/data/lessons.js`;
- `js/core/goals.js`;
- `js/ui/render.js`;
- `js/core/codegen.js`.

Замінити:

- `Lesson` → `Рівень`;
- `Blocks` → `Команди`;
- `Workspace` → `Програма`;
- `Run` → `Запустити`;
- `Reset` → `Очистити`;
- `Code` → `Код`;
- `Task` → `Завдання`;
- `When run` → `Коли запущено`;
- `Next lesson` → `Наступний рівень`.

Критерій готовності: в UI немає англомовних рядків, крім технічних назв у коді.

---

### P0.4. Прибрати небезпечний `instructionHtml`

- [ ] Змінити рівні: `instructionHtml` → `instruction`.
- [ ] Додати renderer, який створює DOM через `textContent`.
- [ ] Дозволити тільки `text`, `strong`, `em`, `br` як structured parts.
- [ ] Додати тести або validator на заборону сирого HTML.

Критерій готовності: тексти рівнів не вставляються через `innerHTML`.

---

### P0.5. Додати execution limits

- [ ] Створити `js/runtime/execution-limits.js` або додати в core.
- [ ] Додати `MAX_EXPANDED_ACTIONS = 500`.
- [ ] Додати `MAX_REPEAT_COUNT = 20`.
- [ ] Додати `MAX_NESTING_DEPTH = 4`.
- [ ] Перевіряти програму до запуску.
- [ ] Показувати дружнє повідомлення при перевищенні.
- [ ] Додати тести.

Критерій готовності: вкладені repeat не можуть зависити сторінку.

---

### P0.6. Додати кнопку Stop

- [ ] Під час виконання `Запустити` замінюється або доповнюється кнопкою `Зупинити`.
- [ ] Runtime має cancel flag або AbortController.
- [ ] Після stop UI повертається в editable state.
- [ ] Показується повідомлення `Програму зупинено.`
- [ ] Додати тест або manual QA-крок.

Критерій готовності: довгу програму можна зупинити.

---

### P0.7. Посилити goal evaluator

- [ ] Додати `exact-path`.
- [ ] Додати `contains-path`.
- [ ] Поточний `path-match` або видалити, або залишити як legacy alias.
- [ ] Додати перевірку порядку кроків.
- [ ] Додати перевірку зайвих кроків.
- [ ] Додати дружні повідомлення.
- [ ] Додати unit-тести.

Критерій готовності: неправильні траєкторії не зараховуються як правильні.

## P1 — доступність і UX

### P1.2. Keyboard shortcuts
