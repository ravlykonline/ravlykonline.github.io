# Ravlyk Artist — навчальний редактор алгоритмічного малювання

`Ravlyk Artist` — браузерна навчальна гра/редактор у стилі Code.org Artist: дитина складає програму з блоків, запускає її, а Равлик малює шлях на полі. Мета проєкту — створити простий, україномовний, безпечний і доступний інструмент для початкової школи без реєстрації та без збору персональних даних.

Цей документ описує, як запустити проєкт, що вже є, що вважається production-версією, і які документи треба читати перед змінами.

---

## 1. Поточний стан проєкту

У проєкті вже є:

- статичний сайт без backend;
- модульна ES-структура у `js/`;
- окремі CSS-файли у `css/`;
- базові рівні у `js/data/lessons.js`;
- базові блоки у `js/data/blocks.js`;
- workspace з блоками та `repeat`;
- запуск програми;
- canvas/SVG-візуалізація шляху Равлика;
- показ згенерованого псевдокоду;
- базові core-тести у `tests/art.core.test.js`.

Важливо: у корені також є старі файли `script.js` і `style.css`. Вони не мають вважатися production-кодом. Їх треба або видалити, або перенести в `legacy/` після перевірки, що в них немає потрібних частин.

---

## 2. Production-вхідні точки

Production-версія має використовувати тільки:

```text
index.html
css/*.css
js/main.js
js/core/*.js
js/data/*.js
js/ui/*.js
tests/*.js
```

Не редагувати `script.js` і `style.css` як основну версію. Якщо треба зберегти їх для історії, перенести:

```text
legacy/script.legacy.js
legacy/style.legacy.css
```

---

## 3. Як запустити локально

Оскільки використовується `type="module"`, сайт треба відкривати через локальний сервер, а не подвійним кліком по `index.html`.

Варіант через Python:

```bash
python -m http.server 8000
```

Потім відкрити:

```text
http://localhost:8000
```

Варіант через Node:

```bash
npx serve .
```

---

## 4. Як запустити тести

Поточні тести можна запускати напряму Node.js:

```bash
node tests/art.core.test.js
```

У майбутньому треба додати `package.json` з командами:

```json
{
  "scripts": {
    "test": "node tests/art.core.test.js",
    "validate:levels": "node tools/validate-levels.js",
    "lint": "eslint js tests",
    "test:e2e": "playwright test"
  }
}
```

Дивись деталі в `TESTING.md`.

## 5. Цільовий функціонал першої стабільної версії

MVP має дати дитині можливість:

1. вибрати рівень;
2. бачити зрозуміле завдання українською;
3. додавати блоки у програму;
4. використовувати повторення;
5. запускати програму;
6. бачити, як Равлик малює шлях;
7. отримувати зрозумілий успіх або підказку;
8. скидати спробу;
9. переходити до наступного рівня;
10. працювати без реєстрації;
11. працювати без збереження персональних даних.

---

## 6. Цільовий функціонал Artist-режиму

Поточна версія керує Равликом через напрямки `вгору/вниз/ліворуч/праворуч`. Це підходить для першого дитячого режиму, але для аналога Code.org Artist треба додати turtle-модель:

- `рухатися вперед`;
- `повернути ліворуч`;
- `повернути праворуч`;
- `повторити`;
- `стрибнути вперед` без малювання;
- `опустити/підняти пензлик` або прихована модель `penDown`;
- `обрати колір`;
- `випадковий колір`;
- пізніше — функції.

Рекомендована прогресія:

1. **Режим клітинок** — найпростіше керування для молодших дітей.
2. **Режим художника** — напрямок, повороти, перо, кольори.
3. **Режим візерунків** — повтори, вкладені повтори, обмеження кількості блоків.
4. **Режим функцій** — пізній етап, не для першого MVP.

## 7. Головні проблеми, які треба виправити першими

P0 — до публічного тестування:

- прибрати або ізолювати legacy-файли `script.js` і `style.css`;
- перекласти весь UI українською;
- посилити перевірку цілей: зараз `path-match` зараховує відвідані клітинки, але не перевіряє порядок і зайві кроки;
- додати ліміт виконання програми, щоб вкладені `repeat` не зависили сторінку;
- додати кнопку `Зупинити` під час виконання;
- зробити клавіатурну альтернативу drag-and-drop;
- прибрати небезпечну залежність від `innerHTML` для текстів рівнів або додати whitelist/sanitizer;
- підготувати PWA/offline-режим.

P1 — до розширення рівнів:

- додати `LEVELS.md`-схему та validator;
- додати undo/redo;
- додати історію станів workspace;
- додати доступний текстовий опис поля;
- зробити `i18n/uk.js`;
- додати Playwright smoke-тести.

P2 — після стабілізації MVP:

- turtle-engine;
- кольори;
- jump/pen up;
- підказки;
- редактор рівнів для вчителя;
- функції.

Повний список задач — у `TASKS.md`.

## 9. Принципи продукту

1. **Дитина має розуміти, що відбувається.** Кожна помилка має пояснюватися людською мовою.
2. **Не має бути страху експериментувати.** Потрібні undo/redo, reset, stop.
3. **Без реєстрації.** Для шкільного використання це сильна перевага.
4. **Без персональних даних.** Не питати ім’я, клас, email.
5. **Offline-first.** Сайт має працювати навіть при нестабільному інтернеті.
6. **Не drag-only.** Усе важливе має працювати кліком, клавіатурою і на планшеті.
7. **Українська мова за замовчуванням.** Усі тексти інтерфейсу й рівнів українською.
8. **Модульність.** Логіка рівнів, engine, UI, state і rendering мають бути розділені.

## 10. Коротка команда для агента

Якщо агент має почати роботу з коду, перші кроки такі:

1. прочитати `AGENTS.md`;
2. перенести legacy-файли у `legacy/`;
3. перекласти UI;
4. додати `package.json`;
5. додати level validator;
6. додати execution limit;
7. додати строгіші goal modes;
8. оновити тести;
9. не ламати існуючий запуск через `index.html` і `js/main.js`.
