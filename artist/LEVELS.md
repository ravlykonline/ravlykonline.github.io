# LEVELS.md — практична специфікація рівнів Ravlyk Artist

Цей документ описує, як мають будуватися рівні, як перевіряти рішення, як додавати нові завдання і яку педагогічну прогресію реалізувати.

---

## 1. Ціль рівнів

Рівні мають навчати дітей базових алгоритмічних понять через малювання:

- послідовність команд;
- напрямок;
- повторення;
- оптимізація кількості блоків;
- передбачення результату програми;
- побудова простих фігур;
- пізніше — кольори, перо, функції.

---

## 2. Проблема поточної реалізації

Поточні рівні лежать у `js/data/lessons.js` і мають поля:

```js
{
  id,
  order,
  title,
  instructionHtml,
  start,
  toolbox,
  success,
  trailColor,
  successMessage,
  successTitle,
  successButton
}
```

Це працює, але є проблеми:

1. `instructionHtml` використовує HTML-рядок. Якщо рівні колись завантажуватимуться не з коду, це XSS-ризик.
2. `success.mode: 'path-match'` перевіряє лише наявність відвіданих клітинок, але не перевіряє порядок і зайві кроки.
3. Текстові обмеження типу “using only 2 blocks” не перевіряються автоматично.
4. UI і рівні англійською.
5. Немає schema validation.
6. Немає педагогічних метаданих: клас, складність, поняття, підказки.

---

## 3. Нова схема рівня

Рекомендована схема:

```js
{
  id: 'grid-01',
  order: 1,
  mode: 'grid',
  title: 'Перша лінія',
  shortTitle: '1',
  description: 'Проведи Равлика вниз на 4 клітинки.',
  instruction: [
    { type: 'text', value: 'Проведи Равлика ' },
    { type: 'strong', value: 'вниз на 4 клітинки' },
    { type: 'text', value: '.' }
  ],
  start: {
    x: 4,
    y: 1,
    dir: 'S'
  },
  toolbox: ['move_s'],
  constraints: {
    maxBlocks: 4,
    maxActions: 10,
    mustUse: [],
    forbidden: []
  },
  success: {
    mode: 'exact-path',
    path: [[4, 2], [4, 3], [4, 4], [4, 5]],
    allowExtraMoves: false
  },
  feedback: {
    successTitle: 'Гарна робота!',
    successMessage: 'Ти намалював рівну лінію.',
    nextButton: 'Наступний рівень',
    hint: 'Спробуй додати чотири команди «вниз».'
  },
  pedagogy: {
    concept: ['sequence'],
    difficulty: 1,
    estimatedAge: '6+',
    teacherNote: 'Перший рівень на послідовність однакових команд.'
  },
  visual: {
    trailColor: 'var(--color-trail-orange)',
    showGuide: true
  }
}
```

## 4. Режими рівнів

### `grid`

Для молодших дітей.

Команди:

- `move_n` — рух вгору;
- `move_s` — рух вниз;
- `move_e` — рух праворуч;
- `move_w` — рух ліворуч;
- `repeat`.

### `turtle`

Для Artist-режиму.

Команди:

- `move_forward`;
- `turn_left`;
- `turn_right`;
- `jump_forward`;
- `set_color`;
- `random_color`;
- `repeat`.

Turtle-рівень має старт:

```js
start: {
  x: 4,
  y: 1,
  heading: 90,
  penDown: true,
  color: '#f5962a',
  strokeWidth: 5
}
```

## 5. Типи перевірки цілі

### 5.1. `minimum-steps`

Для творчого рівня.

```js
success: {
  mode: 'minimum-steps',
  minSteps: 6
}
```

Зараховує, якщо намальовано щонайменше N рухів.

### 5.2. `exact-path`

Для точних завдань у grid-режимі.

```js
success: {
  mode: 'exact-path',
  path: [[4,2], [4,3], [4,4], [4,5]],
  allowExtraMoves: false
}
```

Перевіряє:

- порядок клітинок;
- довжину шляху;
- відсутність зайвих кроків, якщо `allowExtraMoves: false`.

### 5.3. `contains-path`

Для м’якішої перевірки.

```js
success: {
  mode: 'contains-path',
  path: [[4,2], [4,3], [4,4], [4,5]]
}
```

Зараховує, якщо потрібна послідовність зустрічається у шляху.

### 5.4. `exact-segments`

Для Artist/turtle-режиму.

```js
success: {
  mode: 'exact-segments',
  segments: [
    { from: [4,1], to: [4,2] },
    { from: [4,2], to: [4,3] }
  ],
  allowExtraSegments: false
}
```

Перевіряє саме намальовані відрізки.

### 5.5. `contains-segments`

Для складніших малюнків, де порядок може бути неважливий.

```js
success: {
  mode: 'contains-segments',
  segments: [...]
}
```

## 6. Constraints

Обмеження мають бути частиною рівня, а не лише текстом.

```js
constraints: {
  maxBlocks: 2,
  maxActions: 20,
  mustUse: ['repeat'],
  forbidden: ['move_n'],
  maxRepeatCount: 10,
  maxNestingDepth: 2
}
```

### Повідомлення про constraints

- Якщо забагато блоків:

```text
У цьому рівні можна використати не більше 2 блоків. Спробуй застосувати повторення.
```

- Якщо не використано потрібний блок:

```text
У цьому рівні треба використати блок «повторити».
```

- Якщо програма занадто довга:

```text
У програмі забагато кроків. Зменш кількість повторів.
```

## 7. Педагогічна прогресія MVP

### Розділ 1. Послідовність

1. Лінія вниз.
2. Лінія праворуч.
3. Кут.
4. Сходинка.

Ціль: дитина розуміє, що команди виконуються по черзі.

### Розділ 2. Повтори

5. Повторити рух вниз.
6. Повторити рух праворуч.
7. Повторити пару команд.
8. Сходи через repeat.

Ціль: дитина бачить, що repeat скорочує програму.

### Розділ 3. Художник/turtle

9. Рухатися вперед.
10. Повернути праворуч.
11. Намалювати кут через forward + turn.
12. Намалювати квадрат.
13. Намалювати прямокутник.

Ціль: дитина переходить від абсолютних напрямків до напрямку персонажа.

### Розділ 4. Колір і перо

14. Змінити колір.
15. Намалювати дві лінії різного кольору.
16. Стрибнути без малювання.
17. Намалювати розірваний візерунок.

### Розділ 5. Творчість

18. Вільне малювання з мінімальною кількістю кроків.
19. Візерунок з повторенням.
20. Фінальний творчий рівень.

## 9. Валідація рівнів

Потрібен файл:

```text
tools/validate-levels.js
```

Він має перевіряти:

- `id` унікальний;
- `order` унікальний і послідовний;
- `mode` один із `grid`, `turtle`;
- всі блоки з `toolbox` існують;
- `start` відповідає режиму;
- `success.mode` підтримується;
- координати не виходять за поле;
- `constraints.maxBlocks` не менше 1;
- `instruction` не містить сирого HTML;
- `feedback.successTitle` і `feedback.successMessage` заповнені;
- `pedagogy.concept` заповнено.

Запуск:

```bash
node tools/validate-levels.js
```

## 10. Правила написання текстів рівнів

Писати коротко, дружньо, без зайвих термінів.

Добре:

```text
Проведи Равлика вниз на 4 клітинки.
```

Погано:

```text
Згенеруй алгоритмічну послідовність переміщення агента на чотири позиції по вертикальній осі.
```

Правила:

- одна інструкція — одна дія або одна ідея;
- не більше 1–2 речень у базових рівнях;
- якщо є обмеження, показувати його окремо;
- не писати “просто”, “легко”, “очевидно”;
- помилка має підказувати наступну дію, а не сварити.

---

## 11. Підказки

Кожен рівень має мати 1–3 підказки.

```js
hints: [
  'Почни з команди «вниз».',
  'Потрібно пройти 4 клітинки.',
  'Можеш додати 4 однакові команди або використати повторення.'
]
```

Підказки не показуються одразу. Кнопка:

```text
Підказка
```

Після кожного натискання показується наступна.

---

## 12. Відкритість рівнів

Рекомендація для першої версії: усі рівні доступні одразу.

Стан рівнів:

- не пройдений;
- пройдений;
- поточний.

Зберігання:

- у `sessionStorage`, якщо треба зберегти тільки в межах вкладки;
- не в `localStorage` за замовчуванням, бо в школі комп’ютером можуть користуватися різні діти.

---

## 13. Що реалізувати першочергово

1. Замінити `instructionHtml` на структурований `instruction`.
2. Додати `constraints`.
3. Реалізувати `exact-path`.
4. Реалізувати `contains-path`.
5. Реалізувати `exact-segments`.
6. Додати validator рівнів.
7. Перекласти всі поточні рівні українською.
8. Розширити набір до 20 рівнів.
9. Додати hints.
10. Додати педагогічні метадані.
