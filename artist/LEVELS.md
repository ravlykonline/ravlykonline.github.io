# LEVELS.md — практична специфікація рівнів Ravlyk Artist

---

## 1. Ціль рівнів

Рівні навчають дітей базових алгоритмічних понять через малювання:

- послідовність команд → grid-рівні 1–4
- повторення → grid-рівні 3–6
- відносний напрямок (turtle graphics) → turtle-рівні 7+
- кольори і перо → turtle-рівні 14–17
- творчість → фінальні рівні

---

## 2. Поточна реалізація рівнів (після Фази 1)

Файл `js/data/lessons.js` містить 6 grid-рівнів.

Поточна схема:

```js
{
  id: 'lesson-01',       // унікальний рядок
  order: 1,              // порядковий номер
  mode: 'grid',          // 'grid' | 'turtle'
  title: 'Рівень 1: ...',
  instruction: '...',    // plain text, без HTML
  start: { x, y, dir }, // grid: стартова клітинка
  toolbox: ['move_s'],   // доступні блоки
  success: { mode, ... },
  trailColor: 'var(...)',
  successMessage: '...',
  successTitle: '...',
  successButton: '...',
}
```

**Вирішені проблеми Фази 1:**
- ✅ `instructionHtml` → `instruction` (plain text, без XSS)
- ✅ `path-match` → `exact-path` (перевіряє порядок і кількість кроків)
- ✅ Всі тексти українською
- ⬜ `constraints` — ще не реалізовано
- ⬜ `hints` — ще не реалізовано
- ⬜ Schema validation — ще не реалізовано

## 3. Схема рівня (цільова)

```js
{
  id: 'lesson-07',
  order: 7,
  mode: 'turtle',
  title: 'Рівень 7: Художник',
  instruction: 'Намалюй пряму лінію вперед на 100 кроків.',
  start: {
    heading: 0,       // turtle: 0=вгору, 90=вправо
    penDown: true,
    color: 'var(--color-trail-orange)',
    strokeWidth: 4,

  toolbox: ['turtle_forward'],
  success: {
    mode: 'minimum-steps',
    minSteps: 1,
  },
  trailColor: 'var(--color-trail-orange)',
  successMessage: 'Ти намалював пряму лінію.',
  successTitle: 'Чудова робота!',
  successButton: 'Наступний рівень →',
  // [планується]:
  // constraints: { maxBlocks: 4, mustUse: [], forbidden: [] },
  // hints: ['Натисни вперед і запусти.'],
}
```

## 4. Режими рівнів

### `grid`
Клітинкова сітка. Для молодших дітей, перші 6 рівнів.

Команди (internal type → мітка → codegen):
- `move_n` → `вгору` → `рухатися(вгору)`
- `move_s` → `вниз` → `рухатися(вниз)`
- `move_e` → `вправо` → `рухатися(вправо)`
- `move_w` → `вліво` → `рухатися(вліво)`
- `repeat` → `повторити` → `повторити N разів { ... }`

### `turtle`
Вільне малювання. Команди відповідають мові програмування **РАВЛИК**:

| Дія | Internal type | Мітка UI | Параметр | Codegen |
|---|---|---|---|---|
| Рух вперед | `turtle_forward` | `вперед` | steps (default 50) | `вперед 50` |
| Поворот праворуч | `turtle_right` | `праворуч` | degrees (default 90) | `праворуч 90` |
| Поворот ліворуч | `turtle_left` | `ліворуч` | degrees (default 90) | `ліворуч 90` |
| Підняти олівець | `turtle_pen_up` | `підняти` | — | `підняти` |
| Опустити олівець | `turtle_pen_down` | `опустити` | — | `опустити` |
| Колір | `turtle_color` | `колір` | color name | `колір назва` |
| Повтор | `repeat` | `повторити` | count | `повторити N разів { ... }` |

Turtle старт: пікселеві координати відносно центру canvas.

```js
start: {
  heading: 0,         // 0 = вгору
  penDown: true,
  color: 'var(--color-trail-orange)',
  strokeWidth: 4,
}
```

## 5. Типи перевірки цілі

### `minimum-steps` ✅
```js
success: { mode: 'minimum-steps', minSteps: 6 }
```

### `exact-path` ✅
```js
success: {
  mode: 'exact-path',
  goal: [[4,2], [4,3], [4,4], [4,5]],
}
```
Перевіряє порядок і точну кількість кроків.

### `path-match` ✅ (legacy)
Перевіряє лише множину відвіданих клітинок. Залишено для сумісності.

### `exact-segments` ⬜ (P2.7)
```js
success: {
  mode: 'exact-segments',
  segments: [{ from: [0,0], to: [100,0] }],
}
```

## 8. Валідація рівнів (планується, P3)

`tools/validate-levels.js` має перевіряти:
- Унікальні `id` і `order`
- `mode` — `'grid'` або `'turtle'`
- Всі блоки з `toolbox` існують у `blockDefinitions`
- `instruction` — рядок без HTML-тегів
- `success.mode` — підтримуваний режим
- Координати `start` — в межах поля (grid mode)
- `successTitle` і `successMessage` заповнені

---

## 9. Правила написання текстів рівнів

- Коротко, дружньо, без термінів
- Одна дія — одне речення
- Не писати "просто", "легко", "очевидно"
- Помилка підказує наступну дію, не сварить
- Без сирого HTML в `instruction`

Добре:
```text
Намалюй квадрат: рухайся вперед і повертай праворуч чотири рази.
```

Погано:
```text
Виконай алгоритм обходу замкненого контуру з кутом 90°.
```
