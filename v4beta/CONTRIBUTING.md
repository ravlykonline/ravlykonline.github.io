# Contributing

Contributions are welcome, but changes should fit the educational goals and quality bar of the project.

## Before you start

- Open an issue or discussion for large changes before investing heavily in implementation.
- Prefer small, reviewable pull requests over large rewrites.
- Keep user-facing language in Ukrainian unless a file already follows a different convention.

## What to include in a contribution

- a clear problem statement
- focused code changes
- tests for behavior changes when practical
- documentation updates when public behavior, workflow, or policy changes

## Quality expectations

- keep accessibility behavior intact
- avoid breaking public page structure
- preserve encoding safety for Ukrainian content
- do not remove attribution or licensing notices

## Licensing of contributions

By submitting a contribution to this repository, you agree that your contribution may be included in the project under:

- Apache License 2.0 for source code
- the repository's existing content and branding policies for non-code materials

If you submit content or assets that you do not own or cannot license under these terms, do not submit them.
